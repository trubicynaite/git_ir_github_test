Git ir github

Git - kodo versijavimo programa.
Github - versijuoto kodo talpykla internete.

Git komandos

  • init - sukurti git repozitorija dabartiniame aplankale
  • status - pateikia git repozitorijos dabartine info
  • add - prideda nurodytus failus prie commitinimui paruostu failu (stage'ina)
  • rm --cached || restore --staged - isima nurodytus failus is stage'inimo zonos (unstage'ina)
  • commit -m "zinute apie commit'a" - sukuria kodo versija is stageintu failu ir prideda zinute apie pakitimus (pati zinute turetu buti trumpa ir aiski)
  • restore - sugrazina nurodyta/-us faila/-us i pradine busena (jeigu tie failai nebuvo add'inti i stage'ingo zona)
  • push - nusiuncia lokalios repozitorijos commit'us i isorine repozitorija
  • pull - atsiuncia isorines repozitorijos commit'us i lokalia repozitorija
  • log - isveda commit'u sarasa
  • clone - nuklonuoja nurodyta projekta i dabartini aplankala
  • checkout commitID|branchName - pereinama i nurodyta kodo versija arba kita saka
  • branch naujosSakosPavadinimas - sukuriama nauja saka

Branch protection rules

Viena svarbiausiu saku apsaugos taisykliu yra "Pull requestas" (bandymas vienos sakos commitus sukelti i kita saka (merge)).
Pull request taisykles viduje dar reiketu suteikti "Require approvals" nustatyma, kuris reikalauja, kad nurodytas kiekis kitu projektu dalyviu patvirtintu ta pull requesta norint, kad darbas butu apjungtas.

Pull request eiliskumas

Apsiemiau daryti uzduoti. Kokie mano tolimesni veiksmai?

  1. Susikurti nauja saka - git branch newBranch
  2. Nueiti i susikurta saka - git checkout newBranch
  3. Atlieki uzduoti, kuria apsiemei daryti. Su vienu ar keliais commit'ais.
    1. git add .- pridedi pakitimus (taskas prideda visus pakeitimus)
    2. git commit -m "pavadinimas commito"
  4. Nueiti i main/master saka - gut checkout main/master
  5. Atnaujinti main/master saka - git pull
  6. Grizti i savo uzd saka - git checkout newBranch
  7. Sujungti savo saka su main/master saka ir isspresti galimai atsiradusius konfliktus - git merge main
  8. Nusiusti savo darbines sakos pakitimus i github - git push
  9. Githube sukurti pull requesta
  10. Kiti komandos nariai patvirtina arba atmeta pull requesta
  11. Jei pull requestas sulauke pakankamai patvirtinimu - vykdyti merge'a
